# ☁️ R2 Cloud Drive 部署指南

基于 Cloudflare Workers + R2 的免费云端硬盘，采用 Google Material Design 风格，无需本地环境，全程通过网页控制台即可完成部署。

---

## 🛠️ 前置条件

* **Cloudflare 账号**：已开通并登录。
* **绑定支付方式**：Cloudflare 的 R2 服务需要绑定 PayPal 或信用卡（即便在免费额度内，此步骤也为强制要求）。

---

## 🚀 部署步骤

### 第一步：创建 R2 存储桶

1. 登录 [Cloudflare Dashboard](https://dash.cloudflare.com/)。
2. 在左侧边栏点击 **R2**。
3. 点击 **创建存储桶 (Create bucket)**。
4. 输入存储桶名称（例如：`my-cloud-drive`），点击 **创建**。
5. **记录下你的存储桶名称**。

### 第二步：创建 KV 命名空间

1. 进入 **Workers & Pages** 菜单。
2. 点击 **KV** 选项卡，选择 **创建命名空间**。
3. 名称输入：`CLIPBOARD_KV` (注意：名称必须完全一致)，点击 **添加**。
4. 在列表页中，**复制该命名空间的 ID** 并妥善保存。

### 第三步：创建 Workers 实例

1. 在 **Workers & Pages** 页面点击 **创建应用程序 (Create application)**。
2. 点击 **创建 Worker**。
3. 输入名称（例如：`r2-cloud-drive`），点击 **部署**。
4. 部署完成后，点击 **编辑代码 (Edit code)**。
5. 将项目的 `worker.js` 代码粘贴到编辑器中。
6. 点击右上角的 **保存并部署**。

### 第四步：配置绑定与环境变量

在 Worker 的 **设置 (Settings)** -> **变量 (Variables)** 页面中进行以下配置：

| 配置类型 | 变量名称 | 绑定值 / 数值 |
| --- | --- | --- |
| **R2 绑定** | `R2_BUCKET` | 选择你在第一步创建的存储桶 |
| **KV 绑定** | `CLIPBOARD_KV` | 选择你在第二步创建的命名空间 |
| **普通变量** | `SITE_TITLE` | 设置你的网盘名称 (例如：我的云盘) |
| **加密变量** | `ACCESS_PASSWORD` | 设置你的访问密码 |

> **提示**：点击“添加绑定”选择对应的 R2/KV 实例；点击“添加加密变量”以保护你的密码。

### 第五步：完成部署

1. 点击页面上方的 **部署 (Deployments)** 选项卡。
2. 确认最新的部署版本状态为 **Active**。
3. 点击系统分配的 `.workers.dev` 链接即可开始使用。

## 💡 功能亮点

* **多视图切换**：支持网格/列表视图，排序灵活。
* **高效交互**：支持拖拽上传、右键菜单、单击选中与双击操作。
* **跨页面协同**：通过 KV 持久化剪贴板，复制粘贴不丢失。
* **公开共享**：访问 `/shared` 路径可进行无需登录的公开下载。
* **沉浸体验**：支持自动/手动切换深色模式。

## 📂 创建共享文件夹
* **在根目录下直接创建 `shared` 文件夹即可

## ❓ 常见问题 FAQ

> **Q：上传文件后不显示？**
> A：上传成功后，请点击 Snackbar 弹窗中的“刷新”按钮。
> **Q：如何清空剪贴板？**
> A：执行复制或剪切操作后，点击页面底部的“清除”按钮即可。
> **Q：如何自定义域名？**
> A：在 Worker 的 **触发器 (Triggers)** -> **自定义域 (Custom Domains)** 中添加并解析你的域名。
