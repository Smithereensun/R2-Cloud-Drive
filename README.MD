## 功能特性

- 📁 **文件管理**：上传、下载、删除、重命名、新建文件夹
- 🔄 **两种视图**：网格视图 / 列表视图，支持排序
- ✂️ **复制/剪切/粘贴**：通过 D1 持久化剪贴板，跨页面导航不丢失，粘贴完成后自动刷新当前目录
- 🗂️ **D1 路径映射**：R2 对象全部存储在桶根目录，文件夹、复制、剪切、移动、重命名都通过 D1 文件表和目录索引映射虚拟路径
- 🗑 **极速删除**：v1.1.4 重构删除流程，采用 D1 批量操作 + 异步 R2 清理，删除大文件夹速度提升 50-100 倍
- 🔗 **共享文件夹**：`/shared` 路径公开只读共享，无需登录即可下载
- 📦 **大文件分片上传**：大于 512 KiB 的文件自动走分布式存储（优先）或 R2 Multipart Upload 回退，突破单次请求大小限制
- 🌐 **多账号存储节点**：主控账号和外部节点一起参与均衡，超过 512 KiB 的文件按节点容量均衡切片，并优先补齐新增空节点
- 📥 **大文件分段下载**：支持 HTTP Range，网页端大文件会显示进度、实时速度并分段保存
- 🌙 **夜间模式**：深色主题切换，自动记住偏好
- 📊 **容量显示**：默认显示总容量使用情况，点击后展开每个节点的容量明细（每个账号/节点默认 10 GB）
- 🖱️ **右键菜单**：文件/文件夹右键支持复制、剪切、粘贴、下载、重命名、复制链接和删除
- 🎯 **单击选中，双击操作**：单击选中文件，双击下载/进入文件夹
- 📤 **拖拽上传**：支持拖拽文件上传，显示上传进度
- 🧹 **孤儿文件清扫**：未选中文件时点击删除按钮，可扫描并清理 R2 中未被引用的残留对象，释放存储空间
- 👓 **R2 源文件浏览**：存储节点弹窗内可查看各节点（含主控账号）R2 存储桶中的对象列表，支持分页加载
- 🔔 **版本更新检查**：页脚版本号悬停可查看更新状态，自动对比 GitHub 仓库 `version.json`，有新版本时红色角标提示；点击版本号可强制重新检查

## 部署教程

### 前置条件

- [Cloudflare](https://dash.cloudflare.com/) 账号
- [PayPal](https://www.paypal.com/) 你的 cloudflare 账号必须绑定 PayPal，否则无法创建 R2 存储桶

## 部署教程

### 步骤 1：创建 R2 存储桶

1. 登录 [Cloudflare Dashboard](https://dash.cloudflare.com/)
2. 左侧菜单进入 **R2 Object Storage**
3. 点击 **Create bucket**
4. 输入存储桶名称，例如 `my-cloud-drive`
5. 点击 **Create bucket**

### 步骤 2：创建 D1 数据库

1. 左侧菜单进入 **Workers & Pages**
2. 进入 **D1 SQL Database**
3. 点击 **Create database**
4. 输入数据库名称，例如 `r2-cloud-drive-db`
5. 点击 **Create**

### 步骤 3：创建 Worker 并粘贴代码

1. 左侧菜单进入 **Workers & Pages**
2. 点击 **Create application** 或 **Create**
3. 选择 **Worker**
4. 输入 Worker 名称，例如 `r2-cloud-drive`
5. 创建后进入 Worker 的在线编辑器
6. 删除默认示例代码，把本项目的 `worker.js` 全部内容粘贴进去
7. 点击 **Deploy** 保存部署

### 步骤 4：绑定 R2 和 D1

进入刚创建的 Worker 后，按下面方式添加绑定：

1. 打开 **Settings**
2. 进入 **Bindings**
3. 点击 **Add binding**
4. 选择 **R2 bucket**
5. Variable name 填写 `R2_BUCKET`
6. R2 bucket 选择步骤 1 创建的存储桶
7. 保存
8. 再次点击 **Add binding**
9. 选择 **D1 database**
10. Variable name 填写 `DB`
11. D1 database 选择步骤 2 创建的数据库
12. 保存并点击 **Deploy**

> 注意：绑定名称必须完全一致，代码里使用的是 `env.R2_BUCKET` 和 `env.DB`。

> D1 表会由 Worker 首次访问时自动创建，不需要手动执行建表 SQL。

### 步骤 5：设置变量和密钥（可选）

| 变量名                   | 说明                                                                | 是否必须 | 默认值           |
| ------------------------ | ------------------------------------------------------------------- | -------- | ---------------- |
| `ACCESS_PASSWORD`        | 访问密码，不设置则为公开访问                                        | 可选     | 无               |
| `SITE_TITLE`             | 站点标题                                                            | 可选     | `R2 云盘`        |
| `CLOUD_ICON_URL`         | 云盘图标图片链接                                                    | 可选     | 橙色圆角云朵图标 |
| `LOGIN_BACKGROUND_URL`   | 登录页背景图片链接                                                  | 可选     | 淡灰色背景       |
| `STORAGE_NODE_TOKEN`     | 存储节点密钥，节点 Worker 用于接收主控上传分片                      | 可选     | 无               |
| `DOWNLOAD_RANGE_SIZE_MB` | 服务端拼接分布式文件时的内部读取片段大小（MiB），范围 1~32，默认 32 | 可选     | 32               |
| `MAX_UPLOAD_SIZE_MB`     | 单文件最大上传大小（MiB），`0` 或不设置表示不限制                   | 可选     | 不限制           |
| `UPLOAD_SPEED_LIMIT_MBPS`| 浏览器端平均上传限速（MiB/s），`0` 或不设置表示不限制               | 可选     | 不限制           |
| `UPLOAD_API_CONCURRENCY` | 上传接口并发数，控制直传/分片上传请求并发，范围 1~16                | 可选     | 3                |

设置方法：

1. 进入 Worker 的 **Settings**
2. 打开 **Variables and Secrets**
3. 点击 **Add**
4. 选择变量类型：
   - `ACCESS_PASSWORD` 建议选择 **Secret**
   - `STORAGE_NODE_TOKEN` 建议选择 **Secret**
   - `SITE_TITLE`、`CLOUD_ICON_URL`、`LOGIN_BACKGROUND_URL` 可选择普通文本变量
5. 填写变量名和值
6. 点击 **Save**
7. 点击 **Deploy** 让配置生效

### 步骤 6：访问 Worker

部署成功后，在 Worker 页面可以看到一个 `https://你的项目名.你的账号.workers.dev` 地址，打开即可访问云盘。

### 步骤 7：自定义域名（可选）

在 Cloudflare Dashboard → Workers & Pages → 你的项目 → Triggers → Custom Domains 中添加你的域名。

### 大文件上传

网页端上传大于 512 KiB 的文件时会优先尝试分布式上传：主控账号和外部存储节点都会加入分配池，按已用容量/总容量均衡保存分片；如果没有配置外部节点，则分片会落到主控账号。分配采用**平方加权轮询算法**（剩余容量的平方作为权重），新增空节点会因剩余容量大而获得指数级优先分配，快速填补容量差距；主控 R2 只保存 manifest 索引。如果分布式上传失败（如节点不可用），会自动回退到 R2 Multipart Upload。

每个分片最大 90 MiB（Cloudflare Free/Pro 单次请求体限制为 100 MB），分布式上传和 R2 Multipart Upload 都最多支持 10,000 个 part。

可通过 `MAX_UPLOAD_SIZE_MB` 限制单文件最大上传大小；服务端会在直传、R2 Multipart 初始化和分布式上传初始化时拦截超限文件并返回 413。网页端还会读取 `UPLOAD_SPEED_LIMIT_MBPS` 和 `UPLOAD_API_CONCURRENCY`，用于限制上传平均速度和上传请求并发数。

### WebDAV 支持

当前版本不支持 WebDAV 协议，没有实现 `PROPFIND`、`MKCOL`、`MOVE`、`COPY`、`LOCK` 等 WebDAV 方法。请使用内置网页端或 API 访问文件。

### 多账号存储节点

多账号容量合并采用“主控账号 + 多个存储节点 + manifest 索引”：

1. 主控 Worker 负责登录、D1 文件表、manifest 索引、下载拼接。
2. 主控账号和存储节点 Worker 都可以保存大文件分片。
3. 大文件上传时，浏览器把每个分片直接上传到主控账号或不同存储节点。
4. 上传完成后，主控在当前 R2 根目录写入一个 manifest 文件，并在 D1 文件表中映射到目标路径。
5. 下载时主控按 manifest 顺序读取节点分片并拼接返回。

### 大文件下载

`/api/download` 支持 `Range` 请求。网页端下载 512 MiB 及以上文件时，会优先使用浏览器 File System Access API 分段保存文件，并在操作栏显示进度条、已下载大小和实时速度。

为了降低 Cloudflare Worker 单次调用的子请求压力，网页端每 512 MiB 发起一次新的 Range 下载请求；主控 Worker 再按 32 MiB 小段从存储节点读取并校验长度。建议使用 Chrome 或 Edge 下载超大文件；不支持 `showSaveFilePicker` 的浏览器会回退到普通浏览器下载。

添加节点步骤：

1. 在另一个 Cloudflare 账号中按同样方法部署本项目 Worker，并绑定该账号自己的 R2 存储桶。
2. 在该节点 Worker 的 **Variables and Secrets** 中添加 `STORAGE_NODE_TOKEN`，值自定义为一串足够长的密钥。
3. 回到主控 Worker 页面，点击顶部或侧边栏的「存储节点」按钮。
4. 填写节点名称、节点 Worker 地址、节点密钥。
5. 点击「添加节点」后，节点配置会保存到主控的 D1 中。
6. 点击节点右侧的 📂 按钮可查看该节点 R2 中的文件对象，支持分页浏览。

分片分配采用**平方加权轮询**：每次分配分片时，按"剩余容量² ÷ 已分配次数"计算得分，剩余容量 2 倍的节点获得 4 倍权重。同时结合主控 D1 中的节点用量估算，避免节点 R2 容量列表延迟导致连续上传分配不准；新增节点通常占用率最低，因此会大幅优先获得后续分片，直到整体趋于均衡。

注意：manifest 不保存节点密钥；节点密钥保存在主控 D1 中。删除分布式文件时，主控会按 manifest 清理各节点上的分片。

容量显示会把主控账号和所有已添加节点合并计算。默认每个账号/节点按 10 GB 计入总容量；例如主控加 2 个节点时，总容量显示为 30 GB。点击侧边栏底部容量区域可展开查看每个节点的使用情况。

### 共享文件夹

1. 创建根目录下的文件夹 `shared`
2. 在此目录下上传需要公开分享的文件
3. 未登录用户可通过 `https://你的域名/shared` 访问和下载这些文件
4. 共享文件夹支持子目录结构

### 存储路径映射

文件实体都会写入 R2 存储桶根目录，不再用 R2 key 前缀模拟目录。D1 中的文件表负责映射虚拟路径，例如 `shared/1.zip` 可以映射到根目录的 `1.zip`，如果根目录文件名冲突则会生成唯一 storage key。每个目录还会维护独立的 D1 目录索引，列表接口优先读取索引。新建文件夹、复制、剪切、移动、重命名和删除都会同步更新 D1 映射表和目录索引；只有最后一个映射被删除时，才会清理对应 R2 对象或分布式分片。

## 技术栈

- **运行时**：Cloudflare Workers
- **存储**：Cloudflare R2（文件存储）
- **元数据**：Cloudflare D1（文件路径映射、目录索引、剪贴板持久化、节点配置、节点用量估算）
- **前端**：纯 HTML + CSS + JavaScript（Material Design 风格）
- **图标**：Material Icons Round（通过国内 BootCDN 加载，兼容大陆网络）

## 常见问题

**Q：上传或新建文件夹后看不到？**
A：当前版本会维护 D1 目录索引，上传、新建文件夹、删除、移动后列表会更快同步；如果浏览器仍停留在旧页面，可以手动刷新一次。

**Q：复制/粘贴后页面刷新，数据丢失？**
A：本系统使用 D1 存储剪贴板数据，跨页面导航不会丢失。粘贴成功后页面会自动刷新当前目录；如果粘贴按钮不可用，请先执行复制/剪切操作。

**Q：如何清空剪贴板？**
A：复制/剪切后 Snackbar 会出现「清除」按钮，点击即可清除剪贴板数据。

**Q：共享文件放在哪里？**
A：共享文件夹对应 D1 文件表中的 `shared` 虚拟目录；真实 R2 对象仍存储在桶根目录。
